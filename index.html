Remote Forgery Protection by vlado

Rails plugin that automatically adds authenticity token to Ajax request

Remote Forgery Protection is a Rails plugin that automatically adds authenticity token to Ajax requests. Rails protects controller actions from CSRF (Cross-Site Request Forgery) attacks with a token based on a random string stored in the session. The token parameter is named authenticity_token by default and will be embedded in all forms and Ajax requests generated by Rails. What about hand coded Ajax request? You can manually add authenticity_token parameter to all Ajax requests or you can let Remote Forgery Protection plugin do everything for you. Supported Javascript libraries: Prototype, jQuery and ExtJS (let me know if you would like to see it working with some other library)

Install

Install the plugin $ script/plugin install git://github.com/vlado/remote_forgery_protection.git (Optional but recommended) Generate remote_forgery_protection.js file by running $ script/generate remote_forgery_protection
